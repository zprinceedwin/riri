# claude-md-management

Skill: keep `CLAUDE.md` accurate, current, and useful.

## When to use
Onboarding a new repo, after architectural changes, when Claude keeps making the same mistakes that better instructions could prevent.

## Core idea
`CLAUDE.md` is a teammate brief, not a wiki. Capture conventions, gotchas, and "don't do X" rules. Trim anything that has rotted; outdated guidance is worse than none.
