# Triage

First stage of the workflow. Sort incoming work before any commitment.

## Purpose
Classify intent, scope, and urgency. Decide what to drop, defer, or accept.

## Inputs
- Raw request (user prompt, issue, PR, message)
- Existing backlog and in-flight work

## Outputs
- Categorized item: bug / feature / refactor / question / noise
- Estimated effort tier
- Routing target (which agent or skill)

## Exit Criteria
A clear go/no-go decision exists. If "go", the item moves to **contract**.
