# ADRs

Architecture Decision Records.

## When to write one
A choice with long-term consequences that future contributors will need to understand: framework selection, data model, auth approach, deployment topology.

## Template
- **Status**: proposed | accepted | superseded by [ADR-N]
- **Context**: the forces in play
- **Decision**: what we chose
- **Consequences**: what gets easier, what gets harder
- **Alternatives considered**: rejected paths and why

## Naming
`ADRs/ADR-NNNN-<slug>.md` — numbered, never renumbered.
