# learnings

Distilled, reusable insight notes — the durable output of vault-learning.

## What belongs
- Gotchas that bit us once and shouldn't bite us twice
- Validated approaches worth replaying
- Surprising behavior in a library, API, or system
- Patterns we've decided to standardize on

## What doesn't
- Code conventions enforceable by linters
- One-off bug fixes (those live in commits)
- General programming advice (link to source instead)

## Naming
`learnings/<topic-slug>.md` with frontmatter tags.
