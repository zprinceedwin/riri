# PRDs

Product Requirement Docs — the source of truth contracts cite.

## Structure
- **Problem**: who hurts and how
- **Users**: who this is for, what they're trying to do
- **Success metrics**: how we'll know it worked
- **Scope**: in / out
- **Open questions**: unresolved decisions

## Lifecycle
Drafted from a brainstorm (ralph-skills:prd). Locked before contracts spin up. Updated only via explicit amendment, not silent edits.

## Naming
`PRDs/<feature-slug>.md`
